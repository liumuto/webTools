# Git Diff Explainer

用于阅读改动后快速解释“这次到底改了什么，为什么改，风险在哪”。

## 适用场景

- 写提交说明前
- 写 `qa.md` 前
- 代码复查前
- 需要向用户汇报本次修改时

## 输出结构

### 1. 改了什么
- 文件列表
- 每个文件的主要变化

### 2. 为什么改
- 对应需求或 bug
- 方案选择原因

### 3. 影响范围
- 直接影响的页面 / 模块
- 是否可能波及相关交互

### 4. 如何验证
- 建议验证步骤
- 重点边界场景

## 要求

- 以行为变化为中心，不机械复述 diff。
- 说明风险点，但不夸大风险。
- 结论要适合直接用于汇报或整理记录。
