# Android Test Planner

用于为 Android 功能开发或 bug 修复整理测试范围，帮助覆盖状态恢复、权限、机型差异、异常路径与回归风险。

## 输出目标

- 给出最小必要测试范围
- 区分单元测试、集成测试、UI 测试、手工验证
- 明确高风险边界场景

## 测试维度

### 1. 功能主路径
- 用户完成主要任务的标准流程。
- 页面进入、操作、提交、返回是否符合预期。

### 2. 状态恢复
- 旋转屏幕
- 进程重建
- 前后台切换
- 页面返回栈恢复

### 3. 权限与系统能力
- 首次授权
- 拒绝授权
- 永久拒绝后再次引导
- 存储、相机、通知、定位等差异路径

### 4. 网络与数据异常
- 弱网
- 超时
- 空数据
- 服务端异常
- 本地缓存回退

### 5. 设备与展示差异
- 小屏 / 大屏
- 深色模式
- 大字体
- 多语言
- 不同 Android 版本

## 输出格式

- 测试范围
- 必测主路径
- 高风险边界场景
- 建议自动化项
- 本次未覆盖项

## 边界

- 优先围绕当前改动给出测试计划，不泛化成整应用测试文档。
- 未实现自动化时，也应提供可执行的手工验证清单。
