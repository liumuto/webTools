# Android Architecture Guard

用于约束 Android 项目中的架构改动范围，防止为一个需求顺手改 Repository、UseCase、DI、导航或模块边界。

## 使用时机

- 修 bug 时
- 小功能迭代时
- 面对老模块、重依赖模块时
- 用户未明确要求架构升级时

## 判定标准

只有满足以下至少一项，才允许做相关重构：
- 当前结构已无法安全实现本次需求。
- 重复逻辑直接阻碍本次修改。
- 不调整会明显增加本次 bug 风险。
- 当前依赖关系已导致测试或发布阻塞。

## 约束原则

- 优先局部改动，不扩散到无关模块。
- 不把简单页面需求升级成全链路架构改造。
- 不新增只被调用一次的抽象层。
- 保持外部行为与接口契约稳定。

## 特别关注

- Repository / UseCase 是否真的需要新增。
- DI 配置变更是否只影响当前功能。
- 导航图、参数对象、序列化模型改动是否有兼容影响。
- 公共组件抽取是否已被多个场景复用。

## 输出要求

如果发生重构，应说明：
- 为什么必须重构
- 具体范围
- 对哪些模块有影响
- 如何确认行为未变
