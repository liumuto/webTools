# Accessibility Checker

用于检查页面是否对更多用户可访问，特别适合表单型、编辑器型、转换工具型页面。

## 核心检查项

### 1. 语义结构
- 使用正确的语义标签。
- 标题层级连续且清晰。
- 列表、表格、按钮、表单语义正确。

### 2. 表单可用性
- 每个输入控件都应有 `label`。
- 辅助说明与错误提示要能关联到控件。
- 占位符不能替代标签。

### 3. 媒体与图形
- 图片必须有 `alt`。
- 纯装饰图可使用空 `alt`。
- 图标按钮应有可读名称。

### 4. 键盘操作
- 交互元素可通过键盘访问。
- 焦点顺序合理。
- 焦点样式清晰可见。

### 5. 状态表达
- 不只靠颜色表达成功、失败、警告。
- 动态状态变化应能被感知。
- 禁用、加载、展开、选中等状态应明确。

### 6. 阅读体验
- 文本对比度足够。
- 字号、行高、间距保证可读性。
- 长文内容应具备清晰层次。

## 输出要求

输出检查结论时，建议包含：
- 问题位置
- 可访问性风险
- 修改建议
- 优先级

## 边界

- 优先解决明显影响使用的问题。
- 不在未确认需求时引入过度复杂的 ARIA 方案。
