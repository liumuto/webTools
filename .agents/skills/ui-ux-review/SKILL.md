# UI UX Review

用于评审工具页的人机交互质量，重点关注是否清晰、顺手、可预期。

## 关注重点

- 首屏是否能快速理解页面用途
- 输入区、操作区、结果区是否层级清晰
- 主按钮是否足够突出
- 成功、失败、处理中是否有明确反馈
- 长内容、窄屏、空状态是否可用
- 文案是否简洁明确，避免歧义

## 评审维度

### 1. 信息结构
- 页面标题是否说明工具用途。
- 输入、配置、结果是否分区清晰。
- 复杂操作是否有合理分组。

### 2. 操作路径
- 用户完成一次任务需要几步。
- 是否存在不必要点击。
- 默认值是否合理。

### 3. 反馈机制
- 点击后是否有即时反馈。
- 错误提示是否明确说明原因。
- 复制、导出、保存等动作是否有成功提示。

### 4. 可读性
- 标题、正文、说明文字层级是否清晰。
- 颜色对比是否足够。
- 表单标签、占位提示是否易懂。

### 5. 边界场景
- 空输入、超长输入、非法输入是否可处理。
- 小屏、滚动、弹层、长结果区是否仍可用。

## 输出格式

评审结果建议包含：
- 优点
- 主要可用性问题
- 建议优先修复项
- 可后续优化项

## 边界

- 以实用为主，不做脱离项目风格的大改设计。
- 未被要求时，不主动引入框架或复杂设计系统。
