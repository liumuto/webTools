# Android UI State Review

用于评审 Android 页面在加载、成功、失败、空数据、重试、旋转重建等状态下的交互质量。

## 关注重点

- 首屏加载是否明确
- 空态、错误态、无网络状态是否清晰
- 用户操作后是否有即时反馈
- 页面重建后状态是否一致
- 列表、表单、详情页的交互路径是否顺手

## 检查维度

### 1. 页面状态完整性
- 是否明确区分 loading / success / empty / error。
- 是否避免多个状态同时可见造成冲突。
- 重试、刷新、分页加载是否有独立反馈。

### 2. 交互反馈
- 点击、提交、保存、删除、复制、分享是否有明确反馈。
- 禁用态、处理中、成功失败是否可感知。
- 不只依赖颜色表达状态。

### 3. 生命周期与恢复
- 旋转屏幕、进程重建后 UI 状态是否合理恢复。
- 页面返回后是否保留用户需要的上下文。
- 输入中的内容是否意外丢失。

### 4. 列表与表单体验
- 列表滚动回位、分页加载、刷新闪烁是否合理。
- 表单校验是否具体指出错误原因。
- 占位文案不替代标签或标题。

### 5. 可访问性与适配
- TalkBack 是否能读出关键信息。
- 焦点顺序、点击区域、字号缩放是否可用。
- 深色模式、大字体、多语言下布局是否稳定。

## 输出建议

- 当前优点
- 主要状态缺口
- 应优先修复项
- 可后续优化项

## 边界

- 以现有产品交互目标为基准，不无故重设计整页。
- 若问题本质是状态管理错误，应回到 ViewModel / 数据流根因处理。
