# Bugfix First

处理 bug 时，遵循“先定位，后修改；先修问题，再谈优化”。

## 工作原则

1. 先复述问题与复现条件。
2. 阅读相关代码，确认实际执行路径。
3. 找到根因，不凭现象猜测。
4. 仅修改与问题直接相关的代码。
5. 明确验证 bug 是否已消失。

## 优先回答的问题

- 真实错误发生在哪里？
- 是输入问题、状态问题、事件问题还是渲染问题？
- 是否存在更小的修复点？
- 会不会影响现有其他功能？

## 禁止事项

- 借修 bug 顺手重构大片代码。
- 未确认收益前引入新抽象。
- 用兜底逻辑掩盖根因。
- 用大改替代局部修复。

## 输出格式

- bug 现象
- 根因
- 修复点
- 验证结果
