# Android Release Checker

用于 Android 应用在提测、灰度、正式发布前的检查，避免因配置、签名、权限、资源或版本问题导致发布事故。

## 适用场景

- 提测前
- 发版前
- 热修复或紧急补丁发布前
- 多渠道包或多环境配置切换时

## 检查清单

### 1. 版本与构建
- `versionCode`、`versionName` 是否正确。
- release / debug 配置是否切换正确。
- 包名、应用名、环境地址是否符合目标环境。

### 2. 签名与混淆
- 签名配置是否正确。
- 混淆、资源压缩、mapping 输出是否按预期。
- 关键反射、序列化、三方 SDK 是否有 keep 规则。

### 3. 权限与隐私
- 新增权限是否与功能一致。
- 敏感权限申请流程、拒绝处理与说明文案是否完整。
- 隐私弹窗、采集说明、第三方 SDK 合规项是否同步。

### 4. 兼容与稳定性
- 最低系统版本、目标系统版本是否覆盖目标用户。
- 关键流程在主流机型上是否已验证。
- 崩溃监控、日志开关、降级配置是否合理。

### 5. 发布结果说明
- 本次发布改动范围
- 风险点
- 必测项
- 回滚注意事项

## 边界

- 该 skill 用于发布核查，不替代完整测试流程。
- 不在未确认需求时擅自修改发布配置。
