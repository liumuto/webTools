# Frontend Debugger

用于排查原生前端页面问题，优先做根因定位，再给出最小修复方案。

## 适用场景

- 按钮点击无反应
- 事件绑定失效或重复触发
- DOM 更新后界面未同步
- 样式错乱、布局异常、显示闪烁
- 模块间状态不同步
- 页面在部分浏览器下行为异常

## 工作方式

1. 先确认复现路径与用户可见现象。
2. 阅读相关 HTML / CSS / JS 文件，禁止未读先猜。
3. 按“结构、选择器、事件、状态、渲染、浏览器差异”顺序排查。
4. 先找根因，再做最小修改。
5. 输出修复点、影响范围、验证方式。

## 调试清单

### 1. 结构层
- 检查语义结构是否正确。
- 检查目标节点是否真实存在。
- 检查脚本执行时机是否早于 DOM 就绪。

### 2. 选择器层
- 检查 `querySelector` / `querySelectorAll` 是否命中正确节点。
- 检查类名是否与 HTML / CSS / JS 保持一致。
- 检查 DOM 动态更新后旧引用是否失效。

### 3. 事件层
- 检查是否使用 `addEventListener` 绑定。
- 检查是否重复绑定导致多次触发。
- 检查事件代理、冒泡、默认行为是否影响结果。

### 4. 状态层
- 检查输入值、内部变量、界面展示是否同步。
- 检查切换模式后状态是否遗漏重置。
- 检查异步流程是否导致旧状态覆盖新状态。

### 5. 样式层
- 检查类名切换是否生效。
- 检查 flex / grid 布局是否在边界尺寸下溢出。
- 检查滚动容器、定位、层级与可见性问题。

### 6. 浏览器层
- 检查 Chrome / Firefox / Edge 是否存在差异。
- 检查兼容性较弱的 API 是否有替代写法。

## 输出要求

结论应包含：
- 问题现象
- 根因定位
- 最小修复方案
- 验证步骤
- 是否需要同步更新 `qa.md`

## 边界

- 不因调试顺手大规模重构。
- 不添加用户未要求的新功能。
- 若只是单点 bug，优先局部修复而非抽象重写。
