# Android Performance Checker

用于分析 Android 应用的性能问题，优先识别主线程热点、无效渲染、内存浪费与启动路径阻塞。

## 适用问题

- 冷启动慢、首屏白屏久
- 页面滑动掉帧、动画卡顿
- RecyclerView 滚动不流畅
- 频繁 GC、内存抖动、OOM 风险
- 后台任务与 UI 线程竞争

## 检查顺序

1. 明确问题发生阶段：启动、切页、滚动、渲染、后台同步。
2. 阅读相关页面、Adapter、ViewModel、数据加载路径。
3. 先查主线程阻塞，再查无效刷新，再查内存与对象分配。
4. 给出最小性能修复点，不扩大成整体架构重写。

## 核心检查项

### 1. 启动性能
- Application 初始化是否过重。
- 首屏前是否执行了不必要的同步 IO。
- SDK 初始化是否可延后。

### 2. 渲染性能
- 布局层级是否过深。
- 列表 item 是否频繁全量刷新。
- 图片加载、测量、阴影、圆角等是否造成额外开销。

### 3. 线程与任务
- 重计算、JSON 解析、数据库操作是否误跑主线程。
- 协程调度是否合理。
- 是否存在多个重复任务争抢资源。

### 4. 内存与对象分配
- 是否频繁创建临时对象。
- Adapter、监听器、Bitmap、Binding 是否泄漏。
- 缓存策略是否合理。

### 5. 结果输出
- 问题点
- 性能风险等级
- 建议修复点
- 验证指标

## 边界

- 不为了假想瓶颈提前做复杂优化。
- 不把“可读性重构”伪装成性能修复。
