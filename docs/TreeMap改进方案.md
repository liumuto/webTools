
# 持仓资金 Treemap 改进方案

## 1. 布局优化 —— 避免细长矩形

**问题**：部分标的矩形比例过于极端（细长长条），导致阅读困难、文字显示不全。

**改进措施**：

* 使用 **Squarified Treemap（平方化算法）**，保证每个矩形的长宽比接近 1:1。
* 具体逻辑：在计算矩形分区时，优先排列面积相近的元素，避免把大元素挤到一边，小元素被压扁。
* 如果用原生 JS，可以参考 [squarify 算法](https://www.win.tue.nl/~vanwijk/stm.pdf)。

## 2. 统一的间距（Padding / Gap）

**问题**：部分矩形之间紧贴，没有留出黑色边界，视觉不统一。

**改进措施**：

* 在每个矩形绘制时，统一加上 **外间距（margin/padding）**，保证所有矩形之间都有一致的“空隙”。
* 建议使用 **1\~3px 黑色间隔**，既能分隔块状，又不会浪费太多空间。

**CSS 方案**：

```css
.treemap-rect {
  margin: 2px;
  box-sizing: border-box;
  background-clip: content-box; /* 防止边框覆盖颜色 */
}
```

---

## 3. 文本显示策略优化

**问题**：小矩形中的文字溢出、显示不全，影响可读性。

**改进措施**：

* **自适应缩放**：根据矩形面积调整字体大小。
* **动态裁剪**：如果空间不足，优先显示标的简称，金额作为 tooltip 提示。
* **Tooltip 提升信息密度**：鼠标悬停时显示详细信息（全称、金额、占比等）。

**逻辑示例**：

```js
if (rect.width < 80 || rect.height < 40) {
  showShortNameOnly();
} else {
  showFullNameAndValue();
}
```

---

## 4. 增强层级感（分类分组）

**问题**：颜色区分了标的，但资金类别（基金/股票/现金）没有明显分组，缺乏层次感。

**改进措施**：

* 在 Treemap 第一层（类别层级）添加一个 **大矩形框**，包裹该类别下的所有标的。
* 类别框上方显示 **类别名称 + 总金额**，背景色稍微淡化，与内部标的颜色区分开。
* 内部标的继续用不同颜色，保持区分度。

**效果预期**：

* 一眼能看出基金、股票、现金的占比。
* 类别内再对比不同标的的大小。

---

## 5. 交互优化（可选扩展）

* **点击展开/收起**：点击某个类别时，放大该类别区域，方便查看细节。
* **筛选器**：支持按资金类型筛选，仅显示股票/基金/现金。
* **动态排序**：允许根据金额大小排序显示，避免小额标的被分散。

---

## 6. 整体效果目标

改进后的 Treemap 应达到：

* **矩形比例均衡** → 读起来不再有极端细条。
* **间距统一** → 所有方块有规律分隔。
* **文本清晰** → 大标的显示全信息，小标的用缩略 + tooltip。
* **层级分明** → 一眼能区分资金大类，再看类别内细节。

---

👉 最终效果会更接近专业金融软件的 **资金分布热力图**，既有全局观（大类占比），又有细节层次（具体标的）。
