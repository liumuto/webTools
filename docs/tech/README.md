# docs/tech —— 技术方案

存放**具体技术方案**的设计文档。每个方案一篇，基于本目录的通用模板 `_template.md` 撰写。

## 放这里

- 某个特性/问题的技术方案设计
- 选型对比与权衡
- 接口设计、数据结构、流程设计

## 不放这里

- 跨模块的整体架构沉淀 → 放 `../architecture/`
- 计划/排期 → 放 `../plan/`

## 约定

- 新方案：复制 `_template.md` 为 `<方案名>.md` 后填写，**不要直接改 `_template.md`**。
- 方案应包含：背景目标、现状问题、方案概述、详细设计、备选与权衡、风险、验证方式。
