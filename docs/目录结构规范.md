# 📁 项目目录结构规范

## 🔹 统一目录结构

每个模块都应遵循以下统一的目录结构：

```
模块名/
├── docs/                    # 文档目录
│   ├── prd.md              # 产品需求文档
│   ├── technical.md        # 技术文档
│   └── qa.md              # 对话记录
├── data/                   # 数据目录
│   ├── input/             # 输入数据
│   ├── output/            # 输出数据
│   └── export/            # 导出数据
├── css/                    # 样式目录
│   ├── common.css         # 通用样式
│   ├── layout.css         # 布局样式
│   ├── components.css     # 组件样式
│   └── main.css           # 主样式文件
├── js/                     # JavaScript 逻辑目录
│   ├── app.js             # 主应用逻辑
│   ├── components/        # 组件模块
│   ├── utils/             # 工具函数
│   └── modules/           # 功能模块
├── ui/                     # UI 界面目录
│   ├── index.html         # 主页面
│   ├── components/        # 组件页面
│   └── assets/            # 静态资源
│       ├── images/        # 图片资源
│       ├── icons/         # 图标资源
│       └── fonts/         # 字体资源
├── backend/                # 后端代码（如需要）
│   ├── app.py             # Python 后端
│   ├── requirements.txt   # Python 依赖
│   └── config/            # 配置文件
└── README.md              # 模块说明文档
```

## 🔹 目录说明

### 📋 docs/ - 文档目录
- **prd.md**: 产品需求文档，包含功能需求、用户故事、验收标准
- **technical.md**: 技术文档，包含架构设计、API 文档、技术选型
- **qa.md**: 对话记录，记录开发过程中的问题和解决方案

### 📊 data/ - 数据目录
- **input/**: 输入数据文件，如原始数据、配置文件
- **output/**: 输出数据文件，如处理结果、生成数据
- **export/**: 导出数据文件，如报表、统计结果

### 🎨 css/ - 样式目录
- **common.css**: 通用样式，如重置样式、通用类
- **layout.css**: 布局样式，如网格、弹性布局
- **components.css**: 组件样式，如按钮、卡片、表单
- **main.css**: 主样式文件，导入其他样式文件

### ⚙️ js/ - JavaScript 逻辑目录
- **app.js**: 主应用逻辑，初始化、路由、主要功能
- **components/**: 组件模块，可复用的 UI 组件
- **utils/**: 工具函数，数据处理、格式化、验证等
- **modules/**: 功能模块，特定功能的业务逻辑

### 🖥️ ui/ - UI 界面目录
- **index.html**: 主页面，应用入口
- **components/**: 组件页面，独立的页面组件
- **assets/**: 静态资源目录
  - **images/**: 图片资源
  - **icons/**: 图标资源
  - **fonts/**: 字体资源

### 🔧 backend/ - 后端代码（如需要）
- **app.py**: Python 后端主文件
- **requirements.txt**: Python 依赖包列表
- **config/**: 配置文件目录

## 🔹 文件命名规范

### HTML 文件
- 主页面：`index.html`
- 功能页面：`功能名.html`（如 `selectStock.html`）
- 组件页面：放在 `ui/components/` 目录下

### CSS 文件
- 使用小写字母和连字符：`main.css`、`component-name.css`
- 按功能分类：`layout.css`、`components.css`

### JavaScript 文件
- 主文件：`app.js`
- 组件：`组件名.js`（如 `searchBar.js`）
- 工具：`utils.js` 或放在 `utils/` 目录下

### 数据文件
- JSON：`数据名.json`
- CSV：`数据名.csv`
- 按用途分类：`input/`、`output/`、`export/`

## 🔹 迁移指南

1. **保留现有功能**：确保重构过程中不丢失任何功能
2. **更新引用路径**：修改 HTML、CSS、JS 中的文件引用路径
3. **测试验证**：重构后测试所有功能是否正常
4. **更新文档**：同步更新 README.md 和相关文档

## 🔹 注意事项

- 保持向后兼容性，避免破坏现有功能
- 统一使用相对路径引用文件
- 遵循项目的编码规范（BEM、语义化标签等）
- 及时更新 qa.md 记录重构过程
