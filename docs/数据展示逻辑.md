
# 需求文档

## 数据展示样式（自然语言描述）

1. **整体布局**

   * 使用 **矩形网格（Treemap 树图）** 展示资金分布。
   * 整个画布被划分为多个大矩形，每个矩形代表一种资金类型（例如：**现金、基金、股票**）。
   * 每个资金类型矩形内部再细分为小矩形，表示该类型下的具体标的（如某只基金、某个股票、某个银行账户）。

2. **矩形大小**

   * 矩形面积与金额成正比：

     * 类型矩形的面积 = 该类别下所有标的的金额总和。
     * 标的矩形的面积 = 单个标的金额占总体资金的比例。

3. **矩形颜色**

   * 每个标的矩形分配一个 **唯一的颜色**（例如通过固定调色盘或随机生成）。
   * 颜色 **不再表示盈亏**，而仅作为视觉区分。
   * 相同类别下的标的尽量使用 **协调的色系**，便于阅读。

4. **文本显示**

   * **类别矩形**：显示类别名称（如 “基金”）、该类别的总金额。
   * **标的矩形**：显示标的名称、金额（可选显示占比）。
   * 当矩形过小时，可仅显示简称或只显示金额。

5. **交互逻辑（可选）**

   * **悬停**：显示 tooltip，包含标的全称、金额、占比等详细信息。
   * **点击**：支持编辑、删除或查看该标的更多数据。
   * **筛选器**：用户可选择只查看某一类资金（如仅股票）。

---

## 程序逻辑（抽象层）

1. **输入数据结构**

   ```json
   {
     "type": "基金",
     "total": 50000,
     "children": [
       {"name": "招商中证白酒ETF", "value": 20000},
       {"name": "易方达消费ETF", "value": 15000},
       {"name": "南方中证银行ETF", "value": 15000}
     ]
   }
   ```

   * `type`：资金类别
   * `value`：金额（决定矩形大小）

2. **布局算法**

   * 使用有序条带（Strip Treemap）算法，严格按 `value` 分配面积：
     * 顶层：先按类别的 `total`（或 children 和）划分大矩形（类别区域），面积 ∝ 类别总金额；类别区域在画布上自上而下、从左到右铺排；
     * 类别内：在该类别矩形内，对 `children` 按 `value` 降序，采用横向条带逐行铺排；
     * 铺排结果满足：较大的数据更靠左上，较小的更靠右下；不强求方正，允许长条矩形；
     * 面积严格按 `value` 占比计算（总误差仅来源于像素取整）。

3. **渲染规则**

   * 矩形面积 ∝ `value`
   * 矩形颜色 = 唯一标识（调色盘分配）
   * 文本内容 = 名称 + 金额

---

这样的话，你的 **持仓分布图** 就会像拼图一样，清晰展示资金分布，颜色仅作标识，不涉及盈亏。
