# docs —— 项目文档区

本目录是项目所有**文档**的统一入口。代码不放这里，文档不散落在代码目录。

## 子区职责

| 子目录 | 作用 | 典型内容 |
| --- | --- | --- |
| `plan/` | 开发计划 | 路线图、迭代计划、待办拆解 |
| `temp/` | 临时草稿 | 草稿、临时记录；**除 README 外不入库** |
| `architecture/` | 技术架构 | 按模块沉淀的架构图、模块边界、依赖关系 |
| `tech/` | 技术方案 | 具体方案设计；含通用方案模板 `_template.md` |
| `guide/` | 指南 | 脚本使用、开发工作流、环境搭建等 how-to |

## 归档边界

- 一篇文档归属哪个子区，看它的**性质**：计划→plan，架构沉淀→architecture，单点方案→tech，操作指南→guide，未定型草稿→temp。
- 拿不准时先放 `temp/`，定型后再移入对应子区。
- 每个子区都有自己的 `README.md` 说明更细的边界，新增文档前先读对应 README。

## 命名约定

- 文件名用 kebab-case，中文标题可写在文档首行 H1。
- 有时效性的文档在文件名或首行标注日期（YYYY-MM-DD）。
