# 量化选股平台需求文档

## 一、项目背景

为了帮助用户更直观地构建和调试量化选股策略，本系统提供一个基于 **Web 前端（HTML+CSS+JS）** 的可视化界面，结合 **Python 策略引擎** 实现选股逻辑与回测功能。目标是提供 **策略配置 → 运行 → 可视化结果** 的完整闭环。

---

## 二、总体目标

* 前端：提供一个简洁的量化选股界面，用户可调整策略参数、运行策略、查看结果。
* 后端：Python 执行量化策略逻辑（如因子选股、回测、绩效分析），并将结果返回前端展示。
* 数据流：前端负责交互和可视化，后端负责数据运算与逻辑实现。

---

## 三、功能需求

### 1. 策略管理

* **新建策略**

  * 用户可创建一个新的量化策略。
  * 策略参数（如：指标、周期、阈值）可自定义。
* **策略参数配置**

  * 支持输入框、滑块、下拉框等方式调整参数。
  * 参数示例：

    * 选股因子（PE、PB、ROE、动量、MACD、KDJ 等）
    * 策略周期（日、周、月）
    * 股票池（全A股、沪深300、中证500 等）
    * 买入/卖出条件（阈值、交叉点、回撤比例）
* **保存/加载策略**

  * 用户可将配置保存到本地（JSON 格式），再次打开时加载。

### 2. 数据展示

* **股票列表展示**

  * 展示符合条件的股票（表格形式）。
  * 列表包含：代码、名称、选股因子数值、评分等。
* **回测结果展示**

  * 收益曲线图（净值曲线、基准对比）
  * 策略绩效指标（年化收益率、夏普比率、最大回撤）
  * 交易记录（买卖点标注）
* **因子可视化**

  * 可选择单个因子绘制趋势图，方便观察。

### 3. 策略运行

* **前端触发 → 后端计算 → 前端渲染结果**
* **运行按钮**：执行 Python 策略逻辑，返回选股结果。
* **实时调整参数**：修改参数后可立即重新运行。

### 4. 用户交互

* **简洁清晰 UI**：左侧参数配置区，右侧结果展示区。
* **拖拽交互（可选）**：用户拖拽因子组件到策略面板。
* **消息提示**：运行进度、错误提示。

---

## 四、非功能需求

* **性能**：小型数据集（几千只股票 \* 近几年历史数据）可在数秒内返回结果。
* **可扩展性**：支持后续接入更多因子、更多回测功能。
* **跨平台**：前端基于浏览器运行，后端 Python Flask/FastAPI 提供接口。

---

## 五、系统架构

### 1. 技术栈

* 前端：

  * **HTML5 + CSS3 + JavaScript**
  * 可选：ECharts（图表）、Bootstrap/Tailwind（样式）
* 后端：

  * **Python 3.x**
  * FastAPI / Flask 提供 RESTful API
  * Pandas + Numpy + TA-Lib（指标计算）
* 数据存储：

  * 本地 CSV/SQLite（原型阶段）
  * 可升级为 MySQL/PostgreSQL

### 2. 前后端交互

* **前端 → 后端**

  * 请求：`POST /run_strategy`
  * 数据：策略配置 JSON（参数、股票池、时间范围）
* **后端 → 前端**

  * 响应：策略运行结果 JSON

    ```json
    {
      "stocks": [{"code": "600519", "name": "贵州茅台", "score": 0.87}],
      "performance": {
        "annual_return": 0.15,
        "sharpe": 1.2,
        "max_drawdown": -0.08
      },
      "equity_curve": [...],
      "trades": [{"date": "2025-01-05", "action": "BUY", "code": "600519"}]
    }
    ```
